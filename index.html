1. はじめに

本プライバシーポリシー(以下「本ポリシー」)は、おはこめし(以下「本サービス」)におけるユーザーの個人情報の取り扱いについて定めるものです。

開発者はユーザーのプライバシーを尊重し、個人情報の保護に努めます。

2. 収集する情報

本サービスは、以下の情報を収集します。

(a) アカウント情報

  • メールアドレス
  • パスワード(ハッシュ化して保存。開発者はパスワードの平文にアクセスできません)
  • Google アカウントまたは Apple ID の認証識別子(OAuth 認証で使用)

(b) プロフィール情報

  • 表示名(任意)
  • プロフィール画像(任意)
  • 自己紹介文(最大 100 文字、任意)

(c) レシピ・コンテンツ情報

  • レシピ情報(タイトル、材料、作り方、調理時間、人数、美味しさレベル、メモ、参照元 URL)
  • 料理写真(レシピ写真、調理ログ写真)
  • 栄養情報(AI によって推定された値を含む)
  • タグ情報(プリセットタグの選択、カスタムタグの作成)
  • フォルダ情報
  • 調理ログ(日付、写真、メモ)

(d) サブスクリプション情報

  • ご利用プラン(Free / Pro)
  • サブスクリプションの状態(有効 / 解約 / 期限切れ等)
  • RevenueCat 顧客識別子(課金管理のために RevenueCat 社と共有)

(e) ソーシャル情報

  • フォロー関係(フォロー / フォロワー)
  • いいね履歴
  • 家族グループのメンバーシップ情報

(f) AI 処理時のデータ

  • AI レシピ解析時にユーザーが提供した写真(base64 エンコード形式、最大 12MB)
  • AI レシピ解析時にユーザーが入力したテキスト(最大 20,000 文字)
  • AI レシピ解析時にユーザーが入力した URL(最大 2,048 文字)

3. 情報の利用目的

開発者は、収集した情報を以下の目的で利用します。

  • アカウントの認証・セッション管理
  • レシピデータの保存・表示・検索
  • AI レシピ解析(料理写真や文字情報からのレシピ構造の自動生成)
  • コミュニティ機能の提供(レシピの公開、いいね、フォロー)
  • 家族間レシピ共有機能の提供
  • サブスクリプションの管理・利用制限のチェック
  • コンテンツ品質スコアリングによる表示順の最適化
  • サービスの改善・不具合の修正

4. AI 機能におけるデータ処理

  1. AI レシピ解析機能は、Anthropic 社の Claude API を利用しています。
  2. ユーザーが解析を実行した際、提供された写真・テキスト・URL の内容は、Supabase Edge Function を経由して Anthropic 社のサーバーに送信されます。
  3. 送信されたデータは AI 解析の処理にのみ使用され、Anthropic 社の API 利用ポリシーに基づき、モデルのトレーニングには使用されません。
  4. 処理完了後、Anthropic 社のサーバー上にユーザーデータは保持されません(Anthropic 社の API データポリシーに準拠)。
  5. 詳細は Anthropic 社のプライバシーポリシー(https://www.anthropic.com/privacy)をご参照ください。

5. 第三者サービスとのデータ共有

本サービスは、以下の第三者サービスを利用しており、必要な範囲でデータを共有しています。

(a) Supabase(バックエンド基盤)

  • 用途：ユーザー認証、データベース(PostgreSQL)、ファイルストレージ
  • 共有データ：アカウント情報、レシピ・コンテンツ、写真
  • プライバシーポリシー：https://supabase.com/privacy

(b) RevenueCat(サブスクリプション管理)

  • 用途：アプリ内課金の管理、サブスクリプション状態の同期
  • 共有データ：ユーザー識別子、購入履歴、サブスクリプション状態
  • プライバシーポリシー：https://www.revenuecat.com/privacy

(c) Anthropic(AI レシピ解析)

  • 用途：料理写真・テキスト・URL からの AI レシピ解析
  • 共有データ：解析対象の画像・テキスト・URL(一時的な処理のみ)
  • プライバシーポリシー：https://www.anthropic.com/privacy

(d) Google / Apple(認証プロバイダー)

  • 用途：OAuth 認証(Google Sign-In / Apple Sign-In)
  • 共有データ：認証トークン(認証目的のみ。ユーザーのコンテンツデータは共有しません)
  • Google プライバシーポリシー：https://policies.google.com/privacy
  • Apple プライバシーポリシー：https://www.apple.com/legal/privacy/

6. データの保存場所と期間

  1. ユーザーデータは、Supabase のクラウドサーバーに保存されます。
  2. アカウント削除が要求された場合、30 日間の猶予期間を設けた後に、以下のデータを完全に削除します。
    • ユーザーアカウント情報
    • プロフィール情報
    • すべてのレシピ・材料・作り方データ
    • Supabase Storage に保存された画像(レシピ写真、プロフィール画像、調理ログ写真)
    • 調理ログ
    • フォロー関係・いいね履歴
    • サブスクリプション情報
    • 月間使用量データ
  3. 猶予期間中(30 日間)は、アカウントを復元することが可能です。

7. データの安全管理

  1. Row Level Security(RLS)によるデータベースレベルのアクセス制御を実施しています。
  2. ストレージに保存された画像は、RLS により当該ユーザーのみがアップロード・更新・削除できます。
  3. 通信はすべて HTTPS で暗号化されています。
  4. パスワードはハッシュ化して保存しており、開発者を含め平文での閲覧はできません。
  5. サブスクリプションのプラン昇格(Pro 化)はサーバーサイドの Webhook 経由でのみ行われ、クライアント側からの不正な変更はできません。

  • アクセス権：アプリ内でご自身のプロフィール、レシピ、調理ログ等のデータを確認できます。
  • 訂正権：プロフィール編集画面やレシピ編集画面から、ご自身の情報を随時修正できます。
  • 削除権：アプリの設定画面からアカウントの削除を要求できます(30 日間の復元猶予期間あり。猶予期間経過後にすべてのデータが完全に削除されます)。
  • データポータビリティ：将来的にデータのエクスポート機能の提供を予定しています。

  2. 13 歳未満の方が個人情報を提供されたことが判明した場合、速やかに当該情報を削除します。

なお、YouTube や X(Twitter)の URL には現在対応しておりません。AI 解析結果は参考値ですので、必要に応じて編集してください。

ユーザーデータ(レシピ、写真、プロフィール等)は、Supabase のクラウドサーバーに安全に保存されています。データベースレベルのアクセス制御により、ご自身のデータはご本人のみがアクセス・編集できます(公開設定にしたレシピを除く)。

レシピ詳細画面で「おはこ」に指定し、公開設定をオンにすると「みんなのおはこ」に表示されます。

非公開にしたい場合は、同じ画面から公開設定をオフにしてください。非公開にすると、他のユーザーからは閲覧できなくなります。

おはこめしは、日本語、英語、韓国語、中国語(簡体字)に対応しています。端末の言語設定に応じて自動的に切り替わります。
